1. Проблема и решение

Проблема:

Традиционные системы обнаружения вторжений (IDS) основаны на сигнатурах известных атак. Они неэффективны против:

  • Zero-day атак - новых уязвимостей без известных сигнатур
  • Целевых атак (APT) - специально разработанных для конкретной организации
  • Полиморфных вирусов - постоянно меняющих свой код
  • Аномального поведения - действий, которые сами по себе легальны, но в совокупности подозрительны

Решение:

Deep Learning подход: Вместо поиска известных сигнатур, система обучается на примерах нормального трафика и обнаруживает отклонения от нормы. Это позволяет выявлять неизвестные атаки автоматически.

3. Машинное обучение

Deep Autoencoder архитектура:

Входной слой: 13 признаков трафика
Скрытый слой: 8 нейронов (сжатие информации)
Выходной слой: 13 признаков (восстановление)

Как это работает:

  1. Обучение: Модель учится восстанавливать нормальный трафик (ошибка ~0.1-0.3)
  2. Тестирование: На новых данных проверяется ошибка восстановления
  3. Обнаружение: Если ошибка > 0.85 → это аномалия

Математика:

Функция потерь:
L(x) = ||x - decoder(encoder(x))||²

Решение:
Anomaly = True, если L(x) > threshold (0.85)

8. Безопасность

Multi-tenant архитектура:

  • API-аутентификация: Каждый сенсор использует уникальный API-ключ
  • Pydantic валидация: Все входные данные проверяются перед обработкой
  • Полная изоляция: Данные одного клиента недоступны другим
  • Шифрование: API-ключи хранятся в хешированном виде
  • Аудит: Все действия записываются для анализа

Механизм аутентификации:

Запрос: POST /api/sensor_data HTTP/1.1 Authorization: Bearer YOUR_API_KEY Content-Type: application/json { "features": [10.0, 1500.0, 60.0, ...], "packet_count": 10, "total_bytes": 1500 } Ответ: ✓ 202 Accepted - данные приняты ✗ 401 Unauthorized - ключ невалиден ✗ 422 Validation Error - данные некорректны
