# Инструкции по установке и запуску ML-IDS

## Требования системы

### Минимальные требования:
- Python 3.9 или выше
- Операционная система: Windows, Linux или macOS
- Не менее 4 ГБ оперативной памяти
- Не менее 2 ГБ свободного места на диске
- Административные права (для установки зависимостей и запуска сниффера)

### Рекомендуемые требования:
- Python 3.10 или выше
- Не менее 8 ГБ оперативной памяти
- Свободное место на диске: 5-10 ГБ
- Стабильное подключение к интернету

## Подготовка к установке

### 1. Клонирование или скачивание проекта

Если вы клонируете проект из репозитория:
```bash
git clone https://github.com/ваш-аккаунт/ml-ids-project.git
cd ml-ids-project
```

### 2. Создание виртуального окружения (рекомендуется)

```bash
# Создание виртуального окружения
python -m venv venv

# Активация виртуального окружения:
# На Windows:
venv\Scripts\activate
# На Linux/macOS:
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

**ВАЖНО:** На Windows могут возникнуть проблемы с установкой SQLAlchemy из-за проблемы с установкой greenlet. Если у вас возникают ошибки при установке, используйте следующие команды:

```bash
# Для Windows
pip install "SQLAlchemy==1.4.46"
pip install --only-binary=greenlet greenlet
```

## Настройка приложения

### 1. Создание конфигурационного файла

Создайте файл `.env` в корне проекта с настройками:

```
FLASK_SECRET_KEY=your-secret-key-here
DATABASE_URL=sqlite:///site.db
ANOMALY_THRESHOLD=2.5
TRAINING_SAMPLES=500
TIME_WINDOW=2
BPF_FILTER=ip or udp or tcp
TRAIN_DURATION_MINUTES=1
CONTAMINATION=0.05
NN_EPOCHS=50
REDIS_URL=redis://localhost:6379/0
```

### 2. Инициализация базы данных

```bash
python -c "
from app import app, db
from models import User
with app.app_context():
    db.create_all()
    print('База данных создана успешно')
"
```

## Запуск компонентов системы

### 1. Запуск основного сервера

```bash
python app.py
```

Сервер запустится на порту 5000. Откройте в браузере: http://localhost:5000

### 2. Запуск Celery workers

В отдельном терминале:

```bash
celery -A app.celery_app worker --loglevel=info
```

### 3. Запуск Redis (если не установлен как служба)

Убедитесь, что Redis запущен на порту 6379.

### 4. Запуск сенсора

Сначала зарегистрируйтесь в веб-интерфейсе и создайте API-ключ.

```bash
python sensor.py --server localhost --apikey ВАШ_API_КЛЮЧ --iface ИМЯ_ИНТЕРФЕЙСА
```

Для просмотра доступных интерфейсов:
```bash
python check_ifaces.py
```

## Настройка мониторинга

### 1. Регистрация и создание учетной записи
- Откройте http://localhost:5000
- Зарегистрируйтесь или войдите
- Перейдите на вкладку "API Keys" и создайте новый ключ

### 2. Настройка сенсора
- Запустите сенсор с созданным API-ключом
- Убедитесь, что выбираете правильный сетевой интерфейс
- Проверьте, что сенсор подключен к правильному серверу

### 3. Создание и обучение модели
- Перейдите в панель управления (Dashboard)
- Создайте новую модель
- Дайте системе время для сбора обучающих данных
- После обучения активируйте модель для мониторинга

## Устранение неполадок

### Проблемы с установкой зависимостей на Windows:
```bash
pip install --upgrade pip
pip install --only-binary=all tensorflow scikit-learn numpy
```

### Если сенсор не видит интерфейсы:
- Убедитесь, что запускаете с правами администратора
- Проверьте, что Npcap установлен для Windows (https://npcap.com/#download)

### Если данные не обновляются:
- Проверьте, что Celery workers запущены
- Убедитесь, что Redis доступен
- Проверьте API-ключ сенсора

### Если модель не обучается:
- Убедитесь, что сенсор отправляет данные
- Проверьте, что порог обучающих данных достигнут
- Подождите, пока модель не закончит обучение

## Безопасность

### Защита API-ключей:
- Не коммитьте API-ключи в репозиторий
- Регулярно меняйте ключи
- Используйте разные ключи для разных сенсоров

### Обновления безопасности:
- Регулярно обновляйте зависимости
- Проверяйте на уязвимости с помощью:
```bash
pip install safety
safety check
```

## Разработка

### Добавление новых моделей:
- Создайте класс, наследующийся от BaseDetector
- Реализуйте методы train и predict
- Добавьте в список доступных моделей

### Расширение функционала:
- Используйте Celery для фоновых задач
- Сохраняйте данные в базе через SQLAlchemy
- Обновляйте статус в Redis для мгновенного обновления интерфейса

## Остановка системы

1. Остановите сенсор (Ctrl+C)
2. Остановите Celery workers (Ctrl+C)
3. Остановите основной сервер (Ctrl+C)
4. Деактивируйте виртуальное окружение:
```bash
deactivate
```
