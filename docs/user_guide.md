# Полное руководство пользователя ML-IDS

## Обзор системы
ML-IDS (Machine Learning Intrusion Detection System) - это система обнаружения вторжений, основанная на машинном обучении. Она анализирует сетевой трафик в режиме реального времени и обнаруживает аномалии, которые могут указывать на подозрительную активность.

## Архитектура системы

### Компоненты системы:
1. **Сенсор** - захватывает сетевые пакеты с указанного интерфейса
2. **Веб-интерфейс** - предоставляет дашборд для мониторинга и управления
3. **ML-модель** - анализирует трафик на предмет аномалий
4. **База данных** - хранит трафик, модели и настройки
5. **Celery Worker** - обрабатывает задачи машинного обучения
6. **Redis** - используется как брокер задач и кеш

## Начало работы

### Установка и запуск
1. Убедитесь, что Docker и Docker Compose установлены
2. Запустите систему с помощью Docker Compose:
   ```bash
   docker-compose up -d
   ```
3. Дождитесь полной инициализации всех сервисов (около 1-2 минут)

4. Зарегистрируйтесь на сайте http://localhost:5000
   - Используйте действительный email
   - Придумайте надежный пароль

5. Запустите сенсор для захвата трафика:
   ```bash
   python sensor.py --server localhost --apikey <ваш_api_ключ> --iface <имя_интерфейса>
   ```

## Основные функции

### Дашборд (Console)
- **Packets Processing** - отображает количество обработанных пакетов в секунду
- **Network Volume** - объем сетевого трафика в МБ
- **AI Intelligence** - показатель уверенности ML-модели в обнаружении аномалий
- **Core Status** - статус сетевого интерфейса
- **Модели** - список доступных нейросетей с прогрессом обучения
- **График аномалий** - визуализация обнаруженных отклонений от нормы

### Мониторинг трафика (Traffic)
- "Behavioral Flow Analysis" - подробный анализ сетевых соединений
- Отображение Source Node (IP-адрес отправителя) и Target Domain (целевой домен)
- Категоризация соединений (System, Web, etc.)
- Время сессии для каждого соединения

### Обучение модели
- Система накапливает данные для обучения (необходимо 500 образцов)
- При достижении порога автоматически начинается обучение
- Прогресс отображается в реальном времени
- Можно остановить обучение с помощью кнопки "Stop Training"
- После завершения модель становится доступной для активации

### Управление моделями
- Создание новых моделей через "Launch Learning Process"
- Просмотр статуса и прогресса обучения
- Активация моделей для детекции аномалий
- Остановка обучения в процессе

### Управление API-ключами
- Создание новых ключей для сенсоров
- Активация/деактивация ключей
- Удаление устаревших ключей
- Просмотр статистики использования

## Интерфейс пользователя

### Консоль (Dashboard)
- **Deployment Module** - создание новых моделей
- **Active Kernels** - список обученных моделей
- **Anomaly Detection Stream** - график обнаружения аномалий
- **Core System Terminal** - логи системы в реальном времени

### Кнопка "Launch Learning Process"
1. Введите имя модели в поле "Node Identifier"
2. Выберите "Neural Logic" (тип модели)
3. Нажмите кнопку - начнется накопление данных и обучение

### Активация модели
- Нажмите на карточку модели, чтобы активировать её
- Только завершившие обучение модели могут быть активированы
- Активированная модель начинает анализ трафика на аномалии

### Кнопка "Stop Training"
- Появляется только для моделей, которые еще не завершили обучение
- Нажмите кнопку и подтвердите действие
- Обучение будет остановлено и прогресс сброшен

## Работа с сенсором

### Выбор интерфейса
1. Используйте команду для просмотра доступных интерфейсов:
   ```bash
   python sensor.py --list
   ```

2. Выберите интерфейс, который вы хотите мониторить (обычно "Ethernet" или "Wi-Fi")

3. Запустите сенсор с API-ключом:
   ```bash
   python sensor.py --server localhost --apikey <ваш_api_ключ> --iface "Ethernet"
   ```

### Генерация API-ключа
1. Войдите в систему
2. Перейдите в раздел "API Keys" (через навигационное меню)
3. Нажмите "Create New Key"
4. Введите имя сенсора
5. Скопируйте ключ при его создании (больше он не покажется!)

### Проверка активности сенсора
- Сенсор будет выводить информацию о количестве обработанных пакетов
- Индикатор PKTS/SEC показывает скорость обработки
- Данные сразу поступают в веб-интерфейс
- В дашборде можно увидеть активность в реальном времени

## Обнаружение аномалий

### Что анализируется
- Частота соединений с различными доменами
- Объем передаваемых данных
- Временные шаблоны активности
- Повторяющиеся подозрительные паттерны

### Порог срабатывания
- Система использует ANOMALY_THRESHOLD для определения аномалий
- Высокое значение = меньше ложных срабатываний
- Низкое значение = больше чувствительность

### События аномалий
- Появляются в логах системы
- Отображаются в терминале на дашборде
- Вызывают соответствующее событие в интерфейсе

## Устранение неполадок

### Сенсор не отправляет данные
- Проверьте правильность API-ключа
- Убедитесь, что выбран правильный сетевой интерфейс
- Проверьте сетевое подключение к серверу
- Убедитесь, что сервер доступен по адресу

### Обучение не начинается
- Убедитесь, что сенсор активно отправляет данные
- Проверьте, что накоплено не менее 500 образцов
- Убедитесь, что Celery воркер запущен

### Кнопка "Stop Training" не работает
- Убедитесь, что вы вошли в систему
- Кнопка доступна только для моделей с прогрессом < 100%
- Проверьте наличие активной сессии

### График не обновляется
- Проверьте, что сенсор отправляет данные
- Убедитесь, что модель активирована
- Обновите страницу (F5)

### Не удается создать API-ключ
- Проверьте права доступа
- Убедитесь, что сессия активна
