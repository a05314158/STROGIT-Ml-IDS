Научная конференция по кибербезопасности и ML

Март 2026

Проблема

Вызовы современной кибербезопасности:

  • Объем данных: Миллиарды пакетов в день требуют анализа в реальном времени
  • Новые угрозы: Традиционные сигнатурные методы не обнаруживают неизвестные атаки
  • Скорость: Необходимо обнаружение за миллисекунды, а не часы
  • Масштабируемость: Системы должны работать на 1000+ устройств одновременно
  • Ложные срабатывания: Высокий процент false positives перегружает аналитиков

Решение:

Применение Deep Learning для автоматического обнаружения аномалий без предварительного знания сигнатур атак

Архитектура системы

[Сенсор на роутере] ↓ (13 признаков) [REST API + Auth] ↓ (PostgreSQL) [Веб-сервер (Flask)] ↓ (Celery Queue) [ML Worker] ↓ (TensorFlow) [Deep Autoencoder] ↓ [Детекция аномалий]
🔍 Сенсор (sensor.py)

Захват пакетов → Извлечение 13 признаков → HTTP отправка с API-ключом

🌐 Веб-сервер (app.py)

REST API → Pydantic валидация → PostgreSQL хранение → Redis кэширование

🤖 ML Worker (worker.py)

Обучение моделей → Кэширование → Предсказания в реальном времени

Математическая модель

Deep Autoencoder архитектура:

Входной слой (13 признаков) ↓ Скрытый слой (8 нейронов) ↓ Выходной слой (13 признаков)

Функция потерь (Reconstruction Error):

L(x) = ||x - decoder(encoder(x))||²

где x - вектор признаков трафика

Обнаружение аномалии:

Anomaly = True, если L(x) > threshold (0.85)

Нормальный трафик: L(x) ≈ 0.1-0.3
Аномальный трафик: L(x) > 0.85

13 ключевых признаков трафика

1. Энтропия портов

Мера разнообразия используемых портов (Шеннон)

2. Количество пакетов

Общее число пакетов в окне анализа

3. Средний размер пакета

Mean packet size в байтах

4. Стд. отклонение размера

Вариативность размеров пакетов

5. Отношение SYN/ACK

Баланс инициирующих и подтверждающих пакетов

6. Дисперсия интервалов

Вариативность временных промежутков

7. Уникальные IP

Количество различных IP-адресов

8. Энтропия IP

Распределение трафика по IP

9. Количество доменов

Число уникальных доменов

10. Длительность сессии

Время активности в секундах

11. TCP/UDP отношение

Баланс протоколов транспортного уровня

13. Задержка между пакетами

Средний интервал между пакетами

Throughput: 10 req/sec

✅ После оптимизации

2-3 SQL запроса на один HTTP запрос

Время: 50ms

Throughput: 500+ req/sec

Применённые техники:

  • Bulk UPSERT: Вставка 1000+ записей за один запрос вместо N отдельных
  • Model Caching: Загрузка модели один раз, использование из памяти (100x ускорение)
  • Gunicorn Workers: 4 параллельных процесса вместо 1 (50x throughput)
  • PostgreSQL Индексы: Быстрый поиск по доменам и моделям
  • Redis Хеши: Атомарные операции вместо JSON парсинга

Безопасность и изоляция

Multi-tenant архитектура:

  • API-аутентификация: Каждый сенсор использует уникальный API-ключ
  • Pydantic валидация: Все входные данные проверяются перед обработкой
  • Полная изоляция: Данные одного клиента недоступны другим
  • Шифрование ключей: API-ключи хранятся в хешированном виде
  • Аудит-логи: Все действия записываются для анализа

Механизм аутентификации:

Сенсор отправляет: Authorization: Bearer API_KEY Content-Type: application/json Сервер проверяет: 1. Ключ существует? 2. Ключ активен? 3. Принадлежит ли пользователю? Результат: ✓ 202 Accepted - данные приняты ✗ 401 Unauthorized - ключ невалиден ✗ 422 Validation Error - данные некорректны

Преимущество Deep Learning подхода:

  • Обнаруживает неизвестные атаки (zero-day)
  • Адаптируется к новым паттернам трафика
  • Снижает false positives через обучение на нормальном трафике

Результаты и эффективность

Производительность модели:

  • Точность обнаружения: 94-97% на тестовых данных
  • False Positive Rate: 2-3% (значительно ниже сигнатурных методов)
  • Время обнаружения: 5-50ms от момента аномалии
  • Адаптивность: Модель переобучается каждые 24 часа

Сравнение с традиционными методами:

  • Не обнаруживают zero-day
  • Высокий FPR (5-10%)
  • Требуют частые обновления

ML-IDS (Deep Learning)

  • Обнаруживает аномалии
  • Работает на zero-day
  • Низкий FPR (2-3%)
  • Самообучается автоматически
