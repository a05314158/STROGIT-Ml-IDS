Теоретические основы

Математическая модель и архитектура алгоритма

Математическая модель

f(x) = decoder(encoder(x)), где x ∈ R¹³
Encoder:
h = σ(W₁x + b₁)
h ∈ R⁸ (bottleneck layer)
Decoder:
x̂ = W₂h + b₂
x̂ ∈ R¹³ (reconstructed vector)
Loss Function:
Loss = MAE(x, x̂) = (1/n)∑|xᵢ - x̂ᵢ|
Anomaly Detection:
Anomaly = (Loss > τ), где τ = 0.85

Обоснование выбора архитектуры:
  • 13-8-13: Оптимальное соотношение компактности и выразительности
  • ReLU: Позволяет избежать проблемы затухания градиента
  • Dropout: Регуляризация для предотвращения переобучения
  • MAE: Более устойчивая к выбросам по сравнению с MSE

Метрики оценки

Основные метрики:
  • Precision: TP / (TP + FP)
  • Recall: TP / (TP + FN)
  • F1-score: 2 × (precision × recall) / (precision + recall)
  • ROC-AUC: Площадь под ROC-кривой
Дополнительные метрики:
  • Время ответа: Latency на CPU
  • Пропускная способность: Requests/second
  • False Positive Rate: FP / (FP + TN)
  • Memory Usage: RAM consumption
