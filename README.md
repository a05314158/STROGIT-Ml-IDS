# ML-IDS - Machine Learning Intrusion Detection System

## Описание
ML-IDS - это система обнаружения сетевых атак на основе машинного обучения. Система анализирует сетевой трафик в режиме реального времени и выявляет аномалии, которые могут свидетельствовать о попытках взлома или несанкционированного доступа.

## Особенности
- Обнаружение аномалий на основе машинного обучения
- Веб-интерфейс для управления и мониторинга
- Поддержка различных типов ML-моделей (нейронные сети, isolation forest)
- Децентрализованные сенсоры для захвата трафика
- Интерфейс для управления моделями и API-ключами
- Реальное время мониторинга с визуализацией

## Архитектура

### Компоненты системы:
1. **Web-приложение (app.py)**: основной сервер с веб-интерфейсом
2. **Сенсор (sensor.py)**: захватывает трафик с сетевых интерфейсов
3. **Сниффер (sniffer.py)**: многопроцессорный анализ пакетов
4. **Feature Engineering (feature_engineer.py)**: извлечение признаков из пакетов
5. **ML-модели (ml_model.py)**: автоэнкодеры и изоляционные леса
6. **Celery Workers**: асинхронная обработка задач
7. **Redis**: кэширование и хранение статусов

### Поток данных:
1. Сенсор захватывает сетевые пакеты
2. Feature Engineer извлекает признаки из пакетов
3. Сенсор отправляет данные на сервер с аутентификацией
4. Сервер проверяет API-ключ и сохраняет данные
5. Если модель обучена, запускается анализ аномалий
6. Модель анализирует признаки и выдает anomaly score
7. Сервер обновляет статус в Redis
8. Веб-интерфейс отображает актуальные данные

## Установка

### Требования:
- Python 3.9+
- Windows/Linux/macOS

### Установка зависимостей:
```bash
pip install -r requirements.txt
```

### Запуск сервера:
```bash
python app.py
```

### Запуск сенсора:
```bash
python sensor.py --server <адрес_сервера> --apikey <ваш_api_ключ> --iface <имя_интерфейса>
```

### Запуск Celery workers:
```bash
celery -A app.celery_app worker --loglevel=info
```

## Использование

1. Откройте веб-интерфейс (обычно на http://localhost:5000)
2. Зарегистрируйтесь или войдите в систему
3. Создайте API-ключ для сенсора
4. Запустите сенсор на нужном сетевом интерфейсе
5. Создайте и обучите ML-модель
6. Активируйте модель для мониторинга
7. Наблюдайте за статусом и аномалиями в реальном времени

## API-ключи
- Используются для аутентификации сенсоров
- Создаются в панели управления
- Имеют возможности активации, деактивации и удаления

## ML-модели
Система поддерживает два типа моделей:
1. **TFAutoencoderDetector**: автоэнкодер на TensorFlow
2. **IsolationForestDetector**: изоляционный лес

## Функции
- Создание и удаление моделей
- Остановка обучения
- Активация моделей
- Аналитика трафика
- Визуализация аномалий
- Управление сессиями

## Технологии
- **Backend**: Flask, SQLAlchemy
- **ML**: TensorFlow, Scikit-Learn
- **Асинхронные задачи**: Celery
- **Кеширование**: Redis
- **Анализ пакетов**: Scapy
- **Визуализация**: Chart.js
- **Frontend**: HTML/CSS/JS, Bootstrap
