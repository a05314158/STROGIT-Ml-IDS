🚀 Как использовать материалы

Перед конференцией:

  • Прочитайте speaker_notes.md 2-3 раза
  • Откройте conference_presentation.html в браузере
  • Распечатайте conference_handout.html для раздачи
  • Подготовьте ноутбук и проектор
  • Проверьте интернет соединение

Во время доклада:

  • Используйте speaker_notes.md как шпаргалку
  • Показывайте слайды из conference_presentation.html
  • Следите за временем (25 минут на доклад)
  • Используйте ключевые фразы из шпаргалки
  • Готовьтесь к вопросам (ответы в speaker_notes.md)

После доклада:

  • Раздайте conference_handout.html участникам
  • Поделитесь ссылкой на conference_abstract.md
  • Предоставьте контакты и ссылки на GitHub
  • Соберите обратную связь

💡 Ключевые выводы для аудитории

Что нужно запомнить:

  • Deep Learning превосходит сигнатурные методы - обнаруживает неизвестные атаки
  • Autoencoder подход - учится на нормальном трафике, обнаруживает отклонения
  • Производительность критична - 550x улучшение через оптимизацию
  • Multi-tenant архитектура - безопасность и изоляция данных
  • Production-ready - может защищать 1000+ устройств одновременно

Преимущества перед конкурентами:

  • ✅ Обнаружение zero-day атак
  • ✅ 2-3% false positive rate (vs 5-10%)
  • ✅ 5-50ms время обнаружения
  • ✅ Самообучение (переобучение каждые 24 часа)
  • ✅ Масштабируемость (1000+ сенсоров)
