# Архитектура ML-IDS проекта

## Обзор системы

ML-IDS (Machine Learning Intrusion Detection System) - это система обнаружения сетевых атак на основе машинного обучения. Система предназначена для мониторинга, анализа и выявления аномального сетевого трафика в режиме реального времени с использованием адаптивных алгоритмов машинного обучения.

## Компоненты системы

### 1. Веб-приложение (app.py)

**Назначение:**
- Веб-интерфейс для управления системой
- Аутентификация и авторизация пользователей
- Обработка данных от сенсоров
- Управление моделями и API-ключами
- Отображение статусов и аналитики

**Технологии:**
- Flask (веб-фреймворк)
- Jinja2 (шаблонизация)
- SQLAlchemy (ORM)
- Redis (кеширование и статусы)
- Celery (фоновые задачи)

### 2. Сенсор (sensor.py)

**Назначение:**
- Захват сетевых пакетов с указанного интерфейса
- Обработка пакетов через feature engineer
- Отправка данных на сервер через API
- Аутентификация по API-ключу

**Технологии:**
- Scapy (анализ пакетов)
- Requests (HTTP API)
- Multiprocessing (параллельная обработка)

### 3. Сниффер (sniffer.py)

**Назначение:**
- Низкоуровневый захват пакетов
- Многопроцессорная обработка
- Кэширование пакетов в батчах
- Фильтрация и классификация пакетов

**Технологии:**
- Scapy (сниффинг)
- Multiprocessing (параллельная обработка)
- LRUCache (кеширование DNS)

### 4. Feature Engineer (feature_engineer.py)

**Назначение:**
- Извлечение признаков из пакетов
- Вычисление статистических характеристик
- Подготовка данных для ML-моделей
- Обработка временных серий

**Извлекаемые признаки:**
- Количество пакетов
- Общий объем данных
- Медиана длин пакетов
- Энтропия портов
- Процент SYN-запросов
- Процент UDP-пакетов
- Количество уникальных IP-адресов
- Статистика DNS-запросов
- Временные признаки
- Протокольные признаки

### 5. ML-модели (ml_model.py, tf_autoencoder.py)

**Назначение:**
- Обнаружение аномалий в трафике
- Обучение на нормальных данных
- Предсказание аномальных паттернов
- Адаптация к локальному трафику

**Реализованные модели:**
- TFAutoencoderDetector (автоэнкодер на TensorFlow)
- IsolationForestDetector (изоляционный лес)

### 6. Celery Workers (worker.py)

**Назначение:**
- Асинхронная обработка задач
- Обучение ML-моделей
- Предсказание аномалий
- Обработка больших объемов данных

## Архитектурные паттерны

### 1. Микросервисная архитектура
- Веб-сервер
- Сенсоры
- ML-модели
- База данных
- Кеширующие компоненты

### 2. Event-driven архитектура
- Сенсоры отправляют события
- Веб-сервер обрабатывает события
- ML-модели получают события для анализа
- Статусы обновляются в реальном времени

### 3. Pipeline processing
- Сбор данных → Feature engineering → ML анализ → Визуализация
